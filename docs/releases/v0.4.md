## 摄像头不支持感知的通用排查方法

遇到面板或 CLI 显示当前机型暂不支持感知时，不要直接判定硬件不支持。固定分三步：

1. 先确认现象：记录 did、model、`is_online`、`in_use`、`connected`。
2. 再做 direct SDK probe：绕过 denylist 直接验证该型号是否能产出 raw/decoded frame。
3. 最后分流处理：
   - 如果 probe 能出帧，通常是 denylist 误拦截，可以移出 denylist，并重新验证 `connected=true`、进入 `perceive devices`、视觉问答能描述画面。
   - 如果 probe 启动成功但始终无帧，不要继续改 denylist，应转向 LAN、PPCS、Wi-Fi、设备侧实时预览、固件或重配网排查。
   - 如果 native SDK 明确不支持，保留 denylist，或另行设计旁路接入方案。

## Windows 用户快速修复

已确认是 denylist 误拦截后，普通用户可以双击：

```text
docs\scripts\fix-camera-denylist.bat
```

窗口出现后输入摄像头 did 或型号，例如：

```text
1039007350
chuangmi.camera.021a04
```

脚本会自动定位 WSL 中的 Miloco runtime，备份并修改 `camera_extra_info.yaml`，重启服务，启用目标 did，并输出验证状态。
