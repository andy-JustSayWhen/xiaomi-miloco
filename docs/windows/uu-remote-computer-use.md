# UU 远程 + Computer Use 测试注意事项

用途：Codex 通过 Computer Use 插件操作网易 UU 远程，模拟普通 Windows 用户下载、安装和配置 easy-miloco。

## 基本原则

- 鼠标优先。能点击、右键、拖拽、菜单选择，就不要先用键盘硬打。
- 输入文本优先走剪贴板和右键粘贴：URL、API Key、Base URL、OAuth 回调地址都先放到主控端剪贴板，再在远程输入框右键粘贴。
- 不要把远程终端当主通道。UU 远程用于用户视角验证；非视觉部署、日志收集和批量命令应优先走已验证的脚本化通道。
- 每一步看截图确认结果。不要在地址栏、脚本窗口、文件传输窗口里连续盲点。
- 遇到问题先记录到 `docs/windows/validation-record.md`，非硬阻断问题继续完成剩余验收，再统一迭代。

## 外设响应优先级

部署测试时建议：

- 鼠标：主控端优先。
- 键盘：主控端优先。

原因：

- Computer Use 的鼠标点击已经验证可用，适合普通用户视角操作。
- 如果键盘处于被控端优先，主控端输入可能无法进入远程 Edge、资源管理器、安装器和 API 配置框。
- 被控端优先更适合远端本地有人同时操作，或需要避免主控抢输入的场景；不适合作为自动化部署测试默认状态。

检查路径：

```text
UU 远程主窗口 -> 控制中心 -> 外设 -> 键盘 / 鼠标
```

注意：UU 远程的外设菜单可能是悬停展开型。Computer Use 只能稳定点击、右键、拖拽和截图；若二级菜单没有展开，不要反复盲点。先根据现象判断输入链路：鼠标可点但键盘不进远程输入框时，优先怀疑键盘仍是被控端优先。

## 屏幕键盘兜底

如果鼠标可用但主控端键盘不响应，先在远端打开 Windows 屏幕键盘。

鼠标路径：

```text
远端任务栏设置图标 -> 辅助功能 -> 键盘 -> 屏幕键盘
```

已验证：

- 屏幕键盘可以用鼠标向远端输入普通字母。
- 屏幕键盘可以响应组合键。`Ctrl` 键会进入按下/锁定状态，再点目标字母即可发送组合键。
- 在远端 Edge 中，屏幕键盘 `Ctrl` + `L` 可以选中地址栏。

使用建议：

- 长 URL、API Key、Base URL 仍优先用剪贴板 + 右键粘贴。
- 需要组合键时，用屏幕键盘点 `Ctrl` / `Alt` 后再点字母键，比依赖主控端物理键盘更稳定。
- 如果输入框被中文输入法候选条干扰，先用组合键选中位置，再右键粘贴完整内容。

## 安全中转

当远程主窗口右上角信号变红、延迟明显升高、点击发飘、或网络被 VPN/代理影响时，主动打开安全中转。

路径：

```text
UU 远程主窗口 -> 控制中心 -> 更多 -> 使用安全中转 Beta
```

说明：

- 安全中转会牺牲一点实时性，但对部署测试、网页配置、安装器交互通常更稳定。
- 只要不是打游戏，红色信号或高延迟时优先使用安全中转。
- 如果信号是绿色且点击稳定，不需要主动切换。

## 文件传输窗口

UU 文件传输不是远程桌面画面的一部分，Computer Use 可能会看到两个同名 `WIN-home02` 窗口：

- 远程桌面主窗口。
- 文件传输独立窗口。

操作前必须先截图确认当前窗口。不要拿远程桌面窗口句柄去点文件传输窗口。

文件传输只作为兜底：

- 优先通过远程浏览器下载 GitHub Release，保证测试路径接近普通用户。
- 只有远程浏览器下载不可用、且剪贴板/右键粘贴也不可用时，才用文件传输。
- 如果传输速度卡住，停止等待并记录，不要把测试时间耗在传输层。

## Release 部署测试鼠标流程

1. 打开远程浏览器。
2. 主控端剪贴板放入 release zip URL。
3. 鼠标点击地址栏，右键选择粘贴或粘贴并转到。
4. 下载完成后用资源管理器右键解压。
5. 双击 `install.bat`。
6. 安全提示出现时，取消勾选每次询问，再点击运行。
7. 安装器需要输入 API Key、Base URL、模型名、OAuth 回调 URL 时，继续用剪贴板 + 右键粘贴。
8. 安装结束后检查 Miloco WebUI、OpenClaw 入口、账号绑定、模型配置和设备列表。

## 卡住时先判断

| 现象 | 优先判断 | 处理 |
| --- | --- | --- |
| 鼠标能点，键盘不输入 | 键盘可能是被控端优先 | 改键盘主控端优先，或走右键粘贴 |
| 键盘优先级暂时改不了 | 主控键盘链路不可用 | 打开远端屏幕键盘，用鼠标输入或发送组合键 |
| 点击延迟大、信号红 | 网络链路差 | 打开安全中转 |
| 右键粘贴不可用 | 剪贴板未同步 | 记录问题，改用文件传输或用户协助切换设置 |
| 文件传输窗口点不到 | 选错了 UU 窗口 | 切到文件传输独立窗口后再操作 |
| 文件传输 0 KB/s | 传输层卡住 | 停止等待，换浏览器下载或记录阻塞 |
