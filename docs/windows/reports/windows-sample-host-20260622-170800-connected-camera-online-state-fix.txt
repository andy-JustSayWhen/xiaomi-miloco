<windows-sample-host> connected camera online-state fix
Time: 2026-06-22 17:08-17:16 Asia/Shanghai

Problem
- OpenClaw could visually describe camera <camera-did-bedside> / <camera-bedside>, but the chat UI previously showed a top-left node failure for that camera.
- Miloco WebUI /api/miot/scope/cameras data could still mark the same camera as offline even while live/perception streams were connected.

Root cause
- backend/miloco/src/miloco/miot/service.py list_cameras_with_state() used only MiOT cached online && lan_online for is_online.
- backend/miloco/src/miloco/perception/collect/camera_adapter.py _current_source() also exposed cached online && lan_online even for a device already present in the active local stream map.
- When MiOT metadata lagged behind local stream reconnect, the same camera was simultaneously "visible" to perception and "offline" to WebUI/OpenClaw metadata.

Fix
- In MiotService.list_cameras_with_state(), compute cloud_online = online && lan_online, then is_online = cloud_online or did in connected.
- In MiotService.toggle_camera(), use the same connected-aware online rule when enabling cameras, so a connected stream is not rejected because of stale lan_online=false.
- In CameraDeviceAdapter._current_source(), report online=True for devices already in _devices, because that method is only used for active local stream sources.
